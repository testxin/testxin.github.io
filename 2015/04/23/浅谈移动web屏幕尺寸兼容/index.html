移动web屏幕尺寸知识

2015-04-23

　　　
　　　
　 面对如此多的移动端设备屏幕尺寸，你是否被碎片化屏幕搞得晕头转向？你知道屏幕密度是什么吗？ppi和px的关系怎样？开发如何兼容如此多的分辨率？什么样的尺寸设计效果图经济有效？

几个您需要了解的概念

屏幕分辨率：是屏幕图像的精密度，是指显示器所能显示的像素有多少　
像素密度：每英寸像素（英语：Pixels Per Inch，缩写：PPI），又被称为像素密度，是一个表示打印图像或显示器单位面积上像素数量的指数。一般用来计量电脑显示器，电视机和手持电子设备屏幕的精细程度。通常情况下，每英寸像素值越高，屏幕能显示的图像也越精细。
屏幕大小:我们通常所说的iPhone5屏幕尺寸为4英寸、iPhone6屏幕尺寸为4.7英寸，指的是显示屏对角线的长度（diagonal),以英寸（inch）为单位, 1 inch = 2.54cm = 25.4mm。

逻辑分辨率和缩放因子(scale factor between logic point and device pixel)

　看看ios是怎么来处理屏幕尺寸碎片化如下图：
　　
早期的iPhone3GS的屏幕分辨率是320x480(PPI为163），iOS绘制图形均以point(打印和印刷单位)为单位：1 point = 1 px,后来在iPhone4同样大小（依然是3.5寸）的屏幕采用了Retina显示技术，像素密度被放大到2倍，像素分辨率达到960x640（PPI=326）， 显像分辨率提升至iPhone3GS的4倍。但是对于开发者来说，iOS绘制图形的API依然沿袭point为单位。在同样的逻辑坐标系下（320x480，即逻辑分辨率）：1 point = scale*px。简单的来看就如下图：
　
所以在实际效果中3gs和retina的iphone4效果是一样的,这是因为Retina屏幕把2x2个像素当1个像素使用。比如原本24x24像素高的icon，在Retina屏上用了48个像素的高度来显示,导致界面元素都变成2倍大小，反而和3gs效果一样,但是画质却更清晰精致。
ps:安卓采用了同样类似的方案处理屏幕分辨率的碎片化。

总结

因为有神奇的scaler的存在，让我们把屏幕拉到了同一条线上，使得我们在开发中实际只用关注逻辑像素 ，使得我们看到很多著名到UI框架依然只使用了绝对单位px去布局。一点点设计上的小建议，如果您的应用没有特别精致的要求，可以使用iphone5的尺寸来设计。
如果您想正常使用scaler，请一定不要加上<meta name=”viewport” content=”target-densitydpi=device-dpi”> ，特别说明的是，在安卓设备中当 target-densitydpi=device-dpi 时， css中的1px会等于物理像素中的1px.
